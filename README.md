# 教学辅助赋能平台

## 项目概述

基于"可用、可测、可复盘"理念打造的智能教学辅助系统，通过数据采集、智能分析和个性化生成，实现教学全链路赋能。

## 核心特性

### M1 阶段（0-1个月）- 最小可用与数据闭环
- 课堂采集器（语音转写、角色区分）
- 指标库 V1（课前/课中/课后核心指标）
- 基础报告生成（课堂纪要 + 改进清单）
- 数据治理（脱敏、留痕、加密）

### M2 阶段（2-3个月）- 教学增效能力
- 情境导入生成器（PPT与教案自动生成）
- 差异化作业生成器（三层难度）
- 班级学习力可视化雷达图
- 德法迁移力分析

### M3 阶段（4-6个月）- 组织化落地
- 年级/校级数据看板
- 三维联动赋能（学生-教师-学校）
- 校内化部署方案
- 合规审计与A/B测试

## 快速开始

### 前置要求
- Node.js >= 18.x
- Git
- GitHub 账号（用于部署）
- 浏览器：Chrome 85+ / Edge 85+（音频录制功能需要）

### 本地运行

```bash
# 1. 安装前端依赖
cd frontend
npm install

# 2. 安装后端依赖
cd ../backend
npm install

# 3. 启动后端服务（演示模式，无需数据库）
npm run dev
# 后端运行在 http://localhost:5000

# 4. 启动前端服务（新终端）
cd ../frontend
npm run dev
# 前端运行在 http://localhost:3000
```

### 测试账号

```
用户名：teacher
密码：password123
```

### 部署到 GitHub Pages

```bash
# 1. 初始化 Git 仓库
git init
git add .
git commit -m "Initial commit"

# 2. 连接 GitHub（替换 YOUR_USERNAME）
git remote add origin https://github.com/YOUR_USERNAME/teaching-assistant.git
git branch -M main
git push -u origin main

# 3. 启用 GitHub Pages
# 进入仓库 Settings → Pages → Source: GitHub Actions

# 4. 等待自动部署完成（2-3 分钟）
# 访问 https://YOUR_USERNAME.github.io/teaching-assistant/
```

**重要**：GitHub Pages 只能部署前端，后端需要单独部署到 [Railway](https://railway.app/) 或 [Render](https://render.com/)

详细指南：
- 📘 [完整部署指南](./DEPLOYMENT-GUIDE.md)
- 🚀 [快速部署](./QUICK-DEPLOY.md)

## 核心功能模块

### 1. 课堂采集器
- 实时语音转文本
- 师生角色自动识别
- 数据脱敏与加密存储
- 采集开关与状态管理

### 2. 智能分析引擎
- 课前/课中/课后指标计算
- 学习力与迁移力评估
- 互动热区/冷区分析
- 个性化建议生成

### 3. 内容生成器
- 情境导入PPT生成
- 差异化作业生成（三层难度）
- 课堂纪要自动整理
- 改进清单智能提取

### 4. 可视化看板
- 班级级别数据面板
- 年级/校级对比分析
- 趋势变化追踪
- 导出与分享功能

## 数据安全与合规

- **数据加密**: AES-256加密存储
- **访问控制**: 基于角色的权限模型（RBAC）
- **操作留痕**: 完整的审计日志
- **隐私保护**: 自动脱敏敏感信息
- **本地部署**: 支持校内私有化部署
